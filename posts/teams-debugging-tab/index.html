Teams - Debugging Tab.

Introduction

Teams Tab is one of the integration point in Microsoft Teams. It allows a website to be embedded as iframe that was declared in Teams app manifest. It can be added as personal tab, group chat or channels tab inside a team. The tab could also be Teams-context aware using Teams JS SDK.

In this post, I wanted to share about how do you use various common tools to debug Teams Tab application.

Web Developer Tools (Edge/Chrome)

You can open up Teams in your favourite browser of choice especially Edge ðŸ˜‰. Press on F12 and you get access to the developer tools. As usual, navigate to your Teams Tab app and start capturing/viewing the logs.

Make sure you select the right container, in my case it was embedded-page-container (tab).

DevTools on Desktop (Windows/MacOS)

You can also use Teams Desktop to debug your application.

  • Turn on Developer Preview in Teams

  • Right-click Microsoft Teams icon in the desktop tray and click Open DevTools

  • It will launch two new DevTools Windows. One is for the main window and the other one for your tab application and you can debug as usual (it is the same developer tools in Edge/Chrome).

Make sure you select the right container, in my case it was extension-tab-frame (tab).

DevTools on Android

You can also debug your tab app that you deploy in Teams Mobile app. As the tab app has been sideloaded, you can view your app as usual. For debugging the app in Android, you can use the remote debugging feature in Edge/Chrome. For more details, please follow the steps outlined here.
